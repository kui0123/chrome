# Mark2AI 路演页优化方案

生成时间：2026-07-14  
分析对象：`product-roadshow.html`、`extension/README.md`、`extension/manifest.json`、`extension/content/content.js`、`.trae/plans/req-roadshow-feature-gap-plan.md`、`clara-review-product-roadshow-v2.md`、`qa-report-roadshow-2026-07-14.md`

## 1. 总体结论

当前路演页的主定位基本准确：Mark2AI 是一个面向 AI 协作的 Chrome 扩展，用来在网页中直接标记真实 DOM 元素、可视化编辑样式与内容，并导出结构化 Diff 给 AI Agent 执行。这个核心叙事与 `extension/README.md`、`manifest.json` 和真实实现一致。

但页面仍有三个可优化方向。第一，亮点抓取已经覆盖 P0/P1 主能力，但部分亮点的表述权重需要调整，尤其是“11 个样式属性”存在口径风险，“状态持久化”“真实 DOM + 结构化 Diff”“页面内拖拽对齐”应上升为更核心的卖点。第二，页面经过补充后没有严重重复，但存在若干同义重复和层级重复，会让 Step2 与 Step4 显得拥挤。第三，排版设计整体完整但仍偏“功能说明书”，缺少一个足够强的产品演示叙事；建议从“卡片堆叠”改为“场景化演示 + 关键证据 + 轻量说明”的节奏。

## 2. 产品事实基线

从产品文件看，Mark2AI 当前版本为 `2.0.0`。官方扩展名已是 `Mark2AI`，描述为“在 Chrome 中标记 HTML 组件的修改，并生成 diff 文件供 AI Agent 迭代使用”。这意味着路演页使用 “Mark2AI V2.0” 是正确的，不应再混用旧称 “HTML Diff Marker”。

真实功能可以归纳为六组。第一组是标记与选择：用户可以在任意网页上选中元素，生成编号徽章、删除角标和 8 方向把手，并支持 Shift 多选与组合标记。第二组是可视化编辑：包括拖拽移动、调整尺寸、滚轮缩放、滑块精调、颜色、边框、圆角、字体、HTML 内容、链接 href、背景图和图片替换。第三组是 AI 协作输出：导出 Markdown 与 JSON，包含原始 HTML、修改后 HTML、CSS 修改、删除项、选择器和给 AI 的修改说明。第四组是效率体验：工具栏三态切换、右键菜单、快捷键、唤醒按钮、可拖拽面板、最小化面板。第五组是可靠性：`chrome.storage.local` 持久化为主，`sessionStorage` 兜底，刷新或重开后可恢复标记。第六组是视觉系统：V5 UI、四套主题、自定义主题色、Toast、Modal、iOS 风格设置面板、样式隔离。

## 3. 亮点抓取准确性评估

| 当前亮点/表述 | 准确性 | 判断 | 建议 |

| “状态持久化 / 重开恢复” | 准确且被低估 | README v2.0 明确 chrome.storage.local 持久化，真实代码有 save/load。 | 建议作为“可靠性”核心卖点提升，解决用户对误关、刷新丢失的顾虑。 |
| “删除项也进 Diff / 完整 JSON” | 准确但表达需微调 | 删除项进入报告属实，完整 JSON 在报告末尾；但“完整 JSON（节选）”语义冲突。 | 将示例标签改为“JSON 报告示意”或“JSON 节选”，并说明完整 JSON 在报告末尾。 |

### 3.1 遗漏或弱化的真实亮点

第一，状态持久化值得从“补充卡片”上升为信任卖点。路演现场用户会天然担心“刷新后是不是全没了”，而 Mark2AI 的 `chrome.storage.local` 持久化正好能打消顾虑。建议在 Hero 附近增加一句“刷新、误关后也能恢复标记”，或者在工作流末尾加入“可中断、可恢复”的可靠性说明。

第二，样式隔离是当前页面没有充分讲清楚的工程亮点。README v2.0 明确提到 `all: initial + all: unset` 样式隔离，真实价值是避免扩展 UI 被宿主页面 CSS 污染。路演页现在展示了 UI 精致度，但没有解释“为什么在各种网页上都能稳定显示”。建议在交互体验区新增一句“扩展 UI 与宿主页面样式隔离，减少被页面 CSS 干扰”。

第三，右键菜单和快捷键入口可以更场景化。页面已有 Alt + “+” 与三态工具栏，但右键菜单在 README 中也属于使用方式之一。建议不要再新增卡片，而是在“启动方式”小条中统一呈现：工具栏按钮、快捷键、右键菜单。

第四，自定义字体和字体可用性检测是有效亮点，但当前放在 Step2 内部过重。它更适合作为“面板能力的一个例子”，不用独立抢占视觉中心。

### 3.2 可能过度强调或需降权的亮点

“11 个样式属性”不建议继续作为主卖点。它既有数量口径风险，也不是用户最关心的价值。用户真正关心的是“我能把想改的地方讲清楚，AI 不会误解”。建议把数字型卖点替换为能力型卖点。

“四套主题 + 自定义颜色”也不宜放得太靠前。它能体现产品完成度，但不是路演页最强差异点。建议保留在“交互体验”或“设置面板”部分，不放在核心亮点第一屏。

“AI Agent 零歧义”建议弱化绝对化表达。结构化 Diff 能显著降低歧义，但不能保证 AI 永远零歧义。更稳妥的表达是“把歧义降到最低”或“让 AI 能按编号、选择器和修改值执行”。

## 6. 排版设计优化建议

### 6.1 结构与视觉节奏

当前页面大部分区块使用卡片、矩阵和示意面板，信息密度较高，但节奏偏均匀。建议首屏和工作流区增加“主视觉演示”的权重。路演页最需要的是让观众一眼看到“我点了哪个元素、改了什么、AI 收到什么”，而不是先读完很多功能卡片。

具体建议是，Hero 下方加入一个“真实工作台式”的大演示容器：左侧是网页元素被编号标记，右侧是编辑面板，底部是一条导出 Diff 摘要。这样能把 Mark2AI 的核心价值可视化，后面的亮点卡片只做补充说明。

### 6.2 文案层级

页面现在的标题普遍准确，但有些标题偏口号或偏功能清单。建议采用“价值句 + 证据句”的结构。比如 Step2 不写“滑块拖拽 + 页面内直接编辑，所见即所得”，可以改为“先在页面上拖到位，再用面板精调”。正文再解释“位置、尺寸、字体、颜色、HTML 与链接都能进入导出报告”。

Hero 的副标题建议从“扩展做了什么”改为“用户完成了什么”。推荐版本：

> 在任何网页上选中元素，直接改样式、文案、链接和图片；Mark2AI 会把每处修改整理成带编号、选择器和 JSON 的 Diff，交给 AI Agent 精准执行。

### 6.3 视觉风格

当前页面使用柔雾紫、白底卡片、圆角和阴影，整体稳妥，但路演页作为品牌面需要更强的记忆点。建议不要再增加普通卡片，而是把“编号徽章 + 选择器 + Diff 代码”做成独特视觉母题。比如在页面滚动过程中反复出现编号圆点、DOM 选择器短标签、Diff 的 `+ / -` 行，形成 Mark2AI 独有的视觉语言。

同时建议移除 H1 的渐变文字。渐变文字在当前 AI/SaaS 页面里过于常见，也会削弱专业感。可以改成纯色标题，使用主题色只强调关键词或按钮。

### 6.4 交互体验

页面已有主题切换、滑块、开关等可交互元素，但这些交互更像组件展示。建议增加一个轻量的“导出报告预览切换”：点击 Step1/2/3/4 时，右侧报告摘要随步骤更新。这样能让交互服务主叙事，而不是只展示 UI 控件。

移动端方面，需要重点检查工作流示意和 diff 矩阵。当前矩阵列较多，移动端容易拥挤。建议在窄屏下把竞品矩阵改为纵向对比卡，或默认只显示 Mark2AI 与“截图标注 / DevTools / AI IDE”的三张对比卡。

### 6.5 可读性与信息密度

Step2 是当前最拥挤的区域：左侧拖拽示意、中间面板、右侧说明、嵌套字体卡片同时出现。建议拆成“主示意 + 轻说明”，把字体系统、背景图、HTML 编辑这些能力放入一个可展开的“编辑能力清单”。路演讲解时应先让用户理解主流程，再展示完整功能。

## 9. 推荐版本的核心叙事

建议把路演页核心叙事统一为：

> AI 已经能生成网页，但人类反馈仍然常常说不清。Mark2AI 让你在真实网页上直接选中元素，改样式、改文案、改链接、换图片，并把每个修改整理成带编号、选择器、原始/修改后 HTML 和 JSON 的结构化 Diff。你不再需要截图画圈或写长段描述，AI Agent 可以按报告精准执行。

这条叙事比“11 个样式属性”“四套主题”“零歧义”等点状卖点更稳，也更符合真实产品现状。后续页面所有模块都应围绕这条主线展开。
