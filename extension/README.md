# HTML Diff Marker - Chrome 扩展

> **当前版本: v2.0.0**

一个用于在浏览器中标记 HTML 组件、**可视化编辑 CSS 样式和 HTML 内容**，并生成 diff 文件供 AI Agent 迭代使用的 Chrome 扩展。

## 功能特性

### 核心功能

- **组件选择与标记**：点击页面任意 HTML 元素进行标记，显示彩色外框与编号徽章（已选中为绿色、已修改为橙色、多选为紫色虚线）
- **可视化编辑**：拖拽移动、调整大小、滚轮缩放元素大小
- **CSS 样式编辑**：在检查面板中直接编辑位置、尺寸、颜色、边框、圆角等属性
- **HTML 内容编辑**：双击标记的元素可直接编辑其 HTML 文本内容（光标定位在点击位置，不再全选文本）
- **链接地址编辑**：对 `<a>` 标签元素可直接在检查面板中修改 `href` 属性，编辑时不会跳转
- **单个标记删除**：每个标记元素左上角有红色删除角标，点击即可删除该标记
- **实时预览**：所有样式修改立即反映在页面上
- **对齐辅助线**：调整大小和位置时自动显示红色对齐辅助线和尺寸/坐标提示
- **复制当前元素**：在工具栏中可一键复制当前编辑元素的 DOM 并标记为新组件
- **添加新元素**：通过工具栏可直接通过 HTML 代码动态添加新的组件
- **删除当前元素**：从 DOM 中完全移除当前编辑元素
- **Diff 导出**：一键生成包含原始 HTML、修改后 HTML 和 CSS 修改的完整 Markdown 与 JSON 文件
- **快捷键支持**：`Alt+E`（Windows/Linux）/ `⌥+E`（macOS）三态循环切换；`Alt+"+"`（Windows/Linux）/ `⌥+"+"`（macOS）快速进入选择模式

### 标记元素上的交互

在每个标记的元素上，你可以直接进行以下操作：

| 操作 | 方式 | 效果 |
|------|------|------|
| **移动位置** | 拖动元素本体（避开徽章/把手区域） | 改变元素相对位置，显示坐标和位置辅助线 |
| **调整大小** | 拖动边缘紫色圆形把手（8 方向） | 改变元素宽高，显示尺寸和大小对齐辅助线 |
| **滚轮缩放** | `Ctrl/Cmd + 滚轮` | 元素等比缩放 |
| **编辑文案** | 双击元素 | 进入文本编辑模式，光标精确定位在点击位置，Enter 保存，Esc 取消 |
| **打开面板** | 点击右上角绿色/橙色编号徽章，或双击元素 | 弹出完整编辑面板（样式/尺寸/位置/HTML/修改说明） |
| **删除标记** | 点击左上角红色圆形 (x) 按钮 | 删除该标记，恢复原始样式 |
| **编辑链接** |（`<a>` 标签）在编辑面板修改 href 字段 | 修改链接地址，防止编辑时误跳转 |

### 组合标记操作

| 操作 | 方式 | 效果 |
|------|------|------|
| **多选元素** | 选择模式下按住 `Shift` 点击 | 加入/移出多选集（紫色虚线高亮） |
| **创建组合** | 多选后点击浮动工具栏「组合标记」 | 生成组合标记，共享一个紫色外框和 G 编号徽章 |
| **整体移动** | 拖动组合区域 | 所有子元素同步移动，保持相对位置 |
| **整体缩放** | `Ctrl/Cmd + 滚轮` 悬停在组合区域 | 整体视觉缩放（transform: scale） |
| **整体调整大小** | 拖动组合边缘把手（8 方向） | 整体等比缩放 |
| **组合面板** | 点击 G 徽章 | 打开组合编辑面板（缩放滑块/子元素列表/解散/删除） |

## 安装步骤（开发者模式）

1. 打开 Chrome 浏览器，访问 `chrome://extensions/`
2. 右上角开启 **开发者模式** (Developer mode)
3. 点击左上角 **加载已解压的扩展程序** (Load unpacked)
4. 选择本目录（即 extension/ 目录，包含 `manifest.json`）
5. 扩展安装完成，应出现在扩展列表中

## 使用方法

### 快速上手

1. 打开任意网页（`http://` 或 `https://` 页面）
2. **方式 A**：点击浏览器工具栏中的扩展图标（三态循环切换：隐藏 → 唤醒按钮 → 完整工具栏 → 隐藏）
3. **方式 B**：使用快捷键 `Alt+E`（macOS 为 `⌥+E`）三态循环切换；`Alt+"+"`（macOS 为 `⌥+"+"`）快速进入选择模式
4. **方式 C**：在页面右键 → 选择 "开始/停止 选择组件"
5. 在页面上点击需要标记的组件（组件会以紫色虚线高亮）
6. **多选标记**：按住 `Shift` 点击多个元素（紫色虚线高亮），然后点击「组合标记」按钮生成组合标记
7. 标记后自动弹出编辑面板，你可以：
   - **直接拖拽**：鼠标拖动元素改变位置
   - **调整大小**：拖动元素边缘的把手改变尺寸
   - **滚轮缩放**：按住 `Ctrl/Cmd + 滚轮` 缩放元素大小
   - **双击编辑**：双击元素直接编辑 HTML 内容（光标定位在点击位置）
   - **精确编辑**：点击元素上的 `#N` 编号徽章打开检查面板

### 检查面板详解

点击标记元素上的 `#N` 编号徽章，弹出编辑面板包含以下区域：

| 区域 | 功能 |
|------|------|
| **元素信息** | 显示 CSS 选择器，供开发参考 |
| **组件标签** | 简短文字备注，方便快速识别组件用途 |
| **修改说明（给 AI Agent 看）** | 多行文本描述，详细说明本次修改的目的、设计意图，供 AI Agent 理解并应用相同风格 |
| **位置调整** | 左偏移 (left) / 上偏移 (top) 的滑块精确调整（-500 ~ +500px，步长 1px），右侧数值可点击直接编辑 |
| **大小调整** | 宽度/高度的滑块精确调整（10 ~ 2000px，步长 1px），右侧数值可点击直接编辑；支持 px / % 单位切换 |
| **样式编辑** | 颜色、背景色、背景图片、边框、圆角、字体大小等属性；支持上传本地图片作为背景，背景图片编辑区左侧为图片预览和操作按钮（垂直排列：预览图在上，信息按钮在下），右侧为样式重置按钮，可移除背景图；每个属性右侧有 ↺ 重置按钮，可恢复该属性初始值，样式编辑区顶部有「↺ 重置全部」按钮可一键恢复所有样式 |
| **HTML 编辑** | 原始 HTML（只读）和修改后的 HTML |
| **底部操作** | 删除此标记 / 保存修改（面板位置会自动记住，下次打开位置不变） |

### 自定义字体使用说明

**添加自定义字体**：
1. 在检查面板的「字体」下拉框下方，点击「+ 添加自定义字体」按钮
2. 输入字体显示名称（如："我最喜欢的字体"）
3. 输入 CSS font-family 值（如：`"MyFont", sans-serif`）
4. 点击确定后，自定义字体将出现在下拉框的「自定义字体」分组中

**删除自定义字体**：
1. 在下拉框中选择要删除的自定义字体
2. 字体选择器下方会出现「删除此自定义字体」按钮
3. 点击按钮并确认后，该字体将被移除

**字体预览提示（三态）**：
- **失败态（黄色警告）**：当前选择的字体在系统上不可用时显示（包括预设字体和自定义字体）
- **引导态（蓝色信息）**：无自定义字体且未选择字体时的操作引导提示
- **成功态（绿色成功）**：当前选择的自定义字体预览正常时的确认提示

**注意**：
- 自定义字体保存在浏览器本地存储中，清除浏览器数据会导致自定义字体丢失
- 添加的字体在 Diff 中会按用户输入的 font-family 值保存，AI 将依照选项修改

### 工具栏按钮详解

工具栏布局：顶部一排 4 个圆角矩形按钮 + 中间大导出按钮 + 底部图标按钮行

**顶部一排 —— 4 个圆角矩形按钮**
- **选择**：进入/退出选择模式（点击页面元素进行标记）
- **复制**：复制当前编辑元素的 DOM 并作为新的标记组件
- **新增**：弹窗输入 HTML 代码，在当前编辑元素后插入新组件
- **删除**：从 DOM 中完全删除当前编辑的元素及其标记

**中部 —— 导出按钮**
- **导出 Diff**：下载 Markdown 和 JSON 格式的修改报告

**底部 —— 图标按钮行**
- **设置**：打开设置面板，控制编号徽章、拖拽把手、快捷键提示的显示开关
- **清空所有**：清除所有已标记的组件

**计数行**：显示当前标记总数、已修改数（带编辑内容或样式改动的组件）

### 工具栏状态

- **隐藏态**：页面上无任何 UI（初始状态）
- **唤醒态**：页面显示一个小圆按钮（点击展开工具栏）
- **激活态**：完整工具栏显示，可操作

切换方式：点击浏览器扩展图标 **或** 使用快捷键 `Alt+E`（macOS `⌥+E`）三态循环切换；`Alt+"+"`（macOS `⌥+"+"`）快速进入选择模式

### 对齐辅助线详解

当你拖动元素位置或调整大小时，扩展会自动检测与其他标记元素的对齐关系：

- **大小对齐**：当前元素的宽度/高度与其他标记元素相差 ≤ 5px 时
  - 显示红色垂直线（左右边界）/水平线（上下边界）
  - 尺寸浮窗显示 "↔ 200 对齐!" 提示

- **位置对齐**：当前元素的 left/center/right 或 top/center/bottom 与其他元素接近时
  - 显示贯穿整个页面的红色垂直线/水平线
  - 位置浮窗显示 "对齐!" 提示

辅助线帮助你快速将多个元素对齐到统一尺寸或位置，非常适合做设计稿还原。

### 导出 Diff

点击工具栏中的 **导出 Diff** 按钮，下载两个文件：
- `html-diff-<时间戳>.md` - Markdown 格式报告，适合 AI Agent 阅读
- `html-diff-<时间戳>.json` - 结构化数据，便于程序化处理

## 导出文件格式

### Markdown 文件（`.md`）

包含内容：
- 页面信息（URL、标题、时间戳）
- 每个组件的原始 HTML 和修改后 HTML
- CSS 样式修改记录（属性名 + 新值）
- 行级 diff（`-` 表示删除，`+` 表示新增）
- 给 AI Agent 的修改说明
- 完整的 JSON 数据（便于程序化处理）

### JSON 文件（`.json`）

结构化数据，每个标记组件包含：
- `index` - 编号
- `tag` - 元素标签
- `selector` - CSS 选择器
- `originalHTML` - 原始 HTML
- `modifiedHTML` - 修改后的 HTML
- `originalStyles` - 原始样式快照
- `modifiedStyles` - 修改的样式属性字典
- `note` - 组件标签备注
- `description` - 修改说明（给 AI Agent 看）
- `hasChange` - 是否有修改

## 使用 Diff 给 AI Agent

将生成的 `.md` 文件内容直接复制粘贴到对话中，或直接将文件作为附件上传。

**示例提示词**：

```
以下是我对网页组件的修改记录，包含 HTML 结构修改和 CSS 样式修改。
请根据每个组件的原始 HTML 和修改后的 HTML，以及 CSS 样式修改，
还有每个组件的"修改说明"，分析修改意图，保持设计风格，
对项目中相应组件应用同样修改。

[在此粘贴 .md 文件内容]
```

## 注意事项

- 扩展仅在 `http://` 和 `https://` 页面生效，不支持 `chrome://` 等内部页面
- 标记数据存储在 `sessionStorage`，刷新页面后标记保留，关闭标签页后丢失
- 建议及时导出 Diff 文件
- 不要在含有敏感信息的页面使用此扩展
- 组件的 CSS 选择器是自动生成的，仅用于参考
- 拖拽元素时确保点击的是元素本体（不是徽章/把手区域）
- 对 `<a>` 链接元素的点击行为已被拦截，编辑时不会跳转到其他页面
- 编辑面板的位置会在 sessionStorage 中记住，下次打开保持位置不变
- 编辑面板的尺寸（拖拽调整后）和折叠状态也会被记住
- 复制、删除、添加元素等 DOM 结构变更会持久化，刷新页面后仍然保留
- 点击「清空所有标记」会恢复所有被删除的 DOM 元素到原位置
- 非链接元素添加跳转链接后，点击会在新标签页打开；编辑模式下点击不会跳转
- 修改说明支持 Markdown 链接格式 `[链接文字](https://...)`，点击「预览链接」可查看效果
- 部分字体（如幼圆、隶书等）是 Windows 系统自带字体，在 macOS/Linux 上可能不可用，扩展会自动检测并通过三态提示显示字体预览状态（失败态/引导态/成功态）
- 自定义字体保存在浏览器本地存储（chrome.storage.local）中，清除浏览器数据会丢失

## 代码规范（开发者必读）

> **注意：开始任何代码开发任务前，请先完整阅读 [Project_Rule.md](../Project_Rule.md)。**
>
> 以下仅列出最关键的操作规则，完整规范请参考 Project_Rule.md。

### 核心操作规则（严格遵守）

1. **任何文件修改前必须先 Read** —— `Edit` 或 `Write` 的第一步始终是 `Read(目标文件)`。这是工具链的硬性要求，不是建议。
2. **禁止 "DeleteFile + Write" 两步模式** —— Write 工具本身可以覆盖已有文件，不需要先删后写。
3. **禁止创建代码生成器脚本或中转脚本** —— 直接写目标文件本身（`Edit(content.js)` 或 `Write(content.js)`），不经过任何中间脚本。即使 Edit 工具失败，也应该降级到 Write，不应用脚本绕过。
4. **代码与文档必须双向一致** —— 改代码 → 同步检查更新 README.md；改 README.md → 反向检查代码中是否有对应实现；任何新增/删除功能 → 同步更新 README.md 对应章节；manifest.json 配置必须与实际代码行为一致。
5. **禁止提交半成品代码** —— content.js 必须是完整可运行的代码，不能只有 IIFE 开头没有结尾。

### 文件结构说明

- `content.js` 使用 IIFE 包裹，入口函数为 `init()`
- 所有样式类名必须以 `html-diff-marker-` 开头
- 防重复注入标志：`window.__htmlDiffMarkerLoaded`

### 修改后测试

每次修改后请至少执行：
1. `node --check content/content.js` —— 语法检查
2. 在 `test-page.html` 上测试标记/编辑/导出核心流程
3. 确保清理了临时文件

## 开发流程

### Git 分支规范

项目采用简化版 Git Flow 分支管理：

- **main**：生产分支，始终稳定
- **develop**：开发分支，集成所有待发布代码
- **fix/**：Bug 修复分支，从 develop 创建
- **feature/**：功能开发分支，从 develop 创建
- **hotfix/**：紧急修复分支，从 main 创建

### 分支命名规则

| 类型 | 格式 | 示例 |
|-----|------|------|
| Bug 修复 | `fix/<issue-id>-<description>` | `fix/bug-001-select-panel` |
| 功能开发 | `feature/<feature-name>` | `feature/ui-toolbar-redesign` |
| 紧急修复 | `hotfix/<version>` | `hotfix/v1.0.1` |

### 开发步骤

1. 从 develop 创建对应分支
2. 提交代码并推送远程
3. 创建 PR/MR 进行代码审查
4. 审查通过后合并回 develop（使用 --no-ff 保留分支历史）
5. 发布时从 develop 合并到 main

### 合并优先级

紧急修复（hotfix）> Bug 修复（fix）> 功能开发（feature）

### 提交信息规范

| 前缀 | 用途 | 示例 |
|-----|------|------|
| `fix:` | Bug 修复 | `fix: 修复选择模式面板显示问题` |
| `feat:` | 功能开发/UI 迭代 | `feat: 添加深色模式支持` |
| `chore:` | 日常维护/配置更新 | `chore: 更新版本号` |
| `docs:` | 文档更新 | `docs: 更新开发流程说明` |

提交信息应简洁明了，描述具体改动内容。

详细规范请参考 Project_Rule.md 第十四章。

## 故障排除

**问题**：页面上看不到工具栏
**解决**：点击浏览器扩展图标，或使用快捷键 `Alt+E`（macOS `⌥+E`）；`Alt+"+"`（macOS `⌥+"+"`）可快速进入选择模式

**问题**：想修改快捷键
**解决**：访问 `chrome://extensions/shortcuts`，找到 HTML Diff Marker，点击快捷键输入框按下新的组合键即可自定义

**问题**：点击扩展图标没反应
**解决**：确认当前页面是 http 或 https 页面，不是 chrome:// 等内部页面

**问题**：文件下载失败
**解决**：检查浏览器是否允许扩展下载文件权限，可在 `chrome://extensions/` → 扩展详情中设置

**问题**：拖拽/缩放不生效
**解决**：确保元素已被标记（彩色外框或编号徽章），拖拽元素本体而非把手区域

**问题**：元素无法点击（如按钮/链接）
**解决**：被标记的 `<a>` 元素的默认点击行为已被拦截，防止误跳转；通过编辑面板修改 href 即可

**问题**：双击编辑文本不生效
**解决**：确保双击的是元素本体（不是徽章/把手区域），且元素未在选择模式中

**问题**：工具栏按钮"复制当前"/"添加元素"无反应
**解决**：先确保有元素在编辑状态（面板中显示有当前元素编号），复制/删除按钮需要有元素被选中
