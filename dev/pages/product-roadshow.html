PAIN POINTS

AI 生成的网页，修改比做难

五个场景，总有一个让你崩溃过。

1

代码看不懂

AI 给了你一个网页，想微调一下样式。打开代码一看，上千行 HTML/CSS，只能改改文案，元素样式的代码根本看不懂。

"那个按钮的 class 叫 btn-primary，改了会不会影响别的按钮？"
2

描述说不准

截图给 AI，想调整某个区域，却叫不准它的名称，也描述不清想要的颜色和尺寸。

"那个稍微大点儿，这个再弄好看点儿…" — AI 烧了海量 Token 来试图理解，最终结果也只能摸奖。
3

歧义酿恶果

描述有歧义，AI "觉得"它懂了，结果目标区域没变，其他区域被改了。

"截图是 xxxx，用户说的是 xx，我应该按用户说的去执行。" — 原来那个位置叫"它"不叫"Ta"。
4

改得多很苦

10 个元素要调整，建个记事本一条一条写？截图圈一堆红框，最后一起反馈？累。

第 3 条写一半，截图软件退出了。。。。
5

AI 自作主张

描述了，AI 觉得"这样改可能更好"，结果改出来的跟想要的完全不是一回事。

"我根据您的需求进行了优化…" — 谁让你优化的？

每个修改生成带编号、CSS 选择器、原始/修改后 HTML、逐行 Diff 和 JSON 的报告，删除项也单独记录。

标记与修改以 chrome.storage.local 持久化，sessionStorage 兜底，工作流可中断、可恢复。

Diff 导出报告（发给 AI 的版本）

每个修改/删除项包含：编号 · CSS 选择器 · 标签 · 精确修改值 · 原始 HTML · 修改后 HTML · 逐行 Diff · 给 AI 的评论。

修改项 #1 · 主按钮 已修改

选择器：#btn-primary · 修改说明：「这个按钮颜色太淡了，改成主题色，内边距加大到 8px，让它更明显。」

原始：
<button id="btn-primary"
  class="btn">确认</button>

修改后：
<button id="btn-primary"
  class="btn"
  style="background:#70649A;
    color:#fff;padding:8px;">
  确认
</button>

Diff：
- class="btn">确认</button>
+ class="btn"
+   style="background:#70649A;color:#fff;padding:8px;">
+   确认</button>

修改项 #2 · 输入框 已修改

选择器：.form-input · 修改说明：「输入框圆角增大到 8px，边框颜色改为主题色」

原始：
<input type="text"
  class="form-input">

修改后：
<input type="text"
  class="form-input"
  style="border-radius:8px;
    border-color:#70649A;">

Diff：
- class="form-input">
+ class="form-input"
+   style="border-radius:8px;border-color:#70649A;">
