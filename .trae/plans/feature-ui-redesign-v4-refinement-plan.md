# HTML Diff Marker UI v4 精细化优化方案

> **版本**：v4.0（精细化优化版）
> **基于版本**：v3.0 柔雾紫主题（已冻结）
> **状态**：设计确认，待实施
> **核心方向**：从"换色级优化"升级到"元素级精雕"，参考第一版方案的交互与视觉细节

---

## 一、背景与目标

### 1.1 背景

v3 版已完成柔雾紫主题的全面换色和基础组件体系搭建，但在交互细节和视觉精致度上仍有提升空间。用户参考第一版方案的设计，提出了 4 项精细化优化方向。

### 1.2 核心目标

- **交互升级**：位置/大小调整从"输入框+微调按钮"升级为"滑块+数值可编辑"
- **布局优化**：工具栏按钮从"图标+文字竖排"改为"纯文字横排圆角按钮"
- **组件丰富**：增加 iOS 风格开关组件，与复选框分工明确
- **去Emoji化**：移除所有 Emoji 表情，采用 Unicode 符号 + 必要时 SVG 图标的清爽风格
- **设计原则**：每一个元素都要按参考设计的标准精雕细琢，不只是换颜色

## 二、4 项优化细节确认

### 2.1 细节一：滑块交互（位置/大小调整）

#### 设计方案

**整体结构**：属性标签 + 滑块轨道 + 数值显示（点击可编辑）

```
┌──────────────────────────────────────────────────┐
│  位置调整                         ↺ 重置         │
│                                                  │
│  Top                                  20px       │
│  ━━━━━━━━━━━━━━━━●━━━━━━━━━━━━━━━━━             │
│                                                  │
│  Left                                  0px       │
│  ●━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━             │
└──────────────────────────────────────────────────┘
```

#### 视觉规格

| 项 | 规格 |
|----|------|
| **轨道高度** | 4px |
| **轨道背景（未填充）** | `#E5E7EB` |
| **轨道背景（已填充）** | `#8B5CF6`（滑块左侧） |
| **轨道圆角** | 全圆角（9999px） |
| **滑块（Thumb）直径** | 20px |
| **滑块背景** | `#FFFFFF` |
| **滑块边框** | 2px solid `#8B5CF6` |
| **滑块阴影** | `0 2px 6px rgba(139, 92, 246, 0.3)` |
| **滑块 hover** | 阴影加深 + 轻微放大（scale 1.05） |
| **数值显示** | 右侧，12px，`#1F2937`，等宽字体 |
| **数值点击态** | 变为输入框，可精确编辑 |
| **属性标签** | 左侧，12px，`#6B7280`，medium 字重 |

#### 范围与精度

| 属性 | 最小值 | 最大值 | 步长 | 单位 |
|------|--------|--------|------|------|
| X (左偏移) | -500 | +500 | 1 | px |
| Y (上偏移) | -500 | +500 | 1 | px |
| 宽度 | 10 | 2000 | 1 | px |
| 高度 | 10 | 2000 | 1 | px |

#### 交互细节

1. **拖动滑块**：实时更新数值和元素样式
2. **点击数值**：切换为输入框，可键盘输入精确值，失焦或回车后切换回显示态
3. **滚轮调节**：hover 在滑块上时，滚轮可微调（±1px）
4. **重置按钮**：分组右上角，点击后所有属性重置为默认

#### 涉及模块

- 位置调整分组（X / Y 两个滑块）
- 大小调整分组（宽 / 高两个滑块 + 单位切换）
- 组合编辑面板（缩放滑块）

### 2.2 细节二：工具栏按钮布局

#### 设计方案

**结构**：渐变头部 + 一排4个功能按钮 + 底部大导出按钮 + 底部两侧小图标按钮

```
┌──────────────────────────────────────┐
│  HTML Diff Marker      已标记 5 · 已改3 │  ← 渐变头部
├──────────────────────────────────────┤
│                                      │
│  [选择]   [复制]   [新增]   [删除]   │  ← 一排4个圆角按钮
│                                      │
│  [        📤 导出 Diff        ]      │  ← 底部大主按钮
│                                      │
│  ↺ 重置                     ⚙ 设置  │  ← 底部两侧小图标按钮
│                                      │
└──────────────────────────────────────┘
```

#### 4 个功能按钮规格

| 项 | 规格 |
|----|------|
| **按钮数量** | 4个（选择 / 复制 / 新增 / 删除） |
| **按钮文字** | 选择、复制、新增、删除（无 Emoji） |
| **布局** | 等宽横向排列，flex: 1 |
| **高度** | 40px |
| **圆角** | 10px（`--hdm-radius-lg`） |
| **字号** | 12px，字重 500 |
| **间距** | 按钮之间 8px gap |

#### 按钮状态

| 状态 | 背景 | 文字 | 边框 |
|------|------|------|------|
| **默认态** | `#EDE9FE`（浅紫） | `#7C3AED` | 透明 |
| **hover 态** | `#DDD6FE`（稍深） | `#6D28D9` | 透明 |
| **active 态** | `#C4B5FD`（更深） | `#5B21B6` | 透明 |
| **选中/激活态** | `#8B5CF6`（实色） | `#FFFFFF` | 透明 |
| **disabled** | `#F3F4F6` | `#9CA3AF` | 透明 |

> **删除按钮特殊处理**：
> - 默认态：`#FEF2F2` 背景 + `#EF4444` 文字
> - hover 态：`#FEE2E2` 背景 + `#DC2626` 文字
> - 选中态：`#EF4444` 背景 + 白色文字

#### 底部主按钮（导出 Diff）

| **图标** | `📤` 或 SVG 上传/导出图标（待确认） |

### 2.4 细节四：图标方案（去 Emoji 化）

#### 总体原则

- **移除所有 Emoji 表情**，保持清爽简约
- **优先使用 Unicode 符号**（轻量，无依赖）
- **复杂/专业场景使用 SVG 图标**（线性风格，统一 16px/18px/20px）

#### Unicode 符号清单（常用）

| 用途 | 符号 | 说明 |
|------|------|------|
| 重置/刷新 | ↺ | 顺时针箭头 |
| 设置/齿轮 | ⚙ | 齿轮 |
| 关闭/取消 | ✕ | 乘号（注意：不是 Emoji） |
| 确认/成功 | ✓ | 对勾 |
| 警告/注意 | ⚠ | 感叹号三角 |
| 信息/提示 | ℹ | 信息i |
| 上移/增加 | ↑ / + | 箭头或加号 |
| 下移/减少 | ↓ / - | 箭头或减号 |
| 左移 | ← | 左箭头 |
| 右移 | → | 右箭头 |
| 复制 | ⎘ / ❐ | 复制/文档 |
| 删除 | 🗑 太Emoji？用 ✕ 或文字 | 待定 |
| 选择/光标 | ◎ / ○ | 圆形靶心 |
| 添加/新增 | + | 加号 |
| 导出/上传 | 📤 太Emoji？用 ↑ 或文字 | 待定 |
| 导入/下载 | 📥 太Emoji？用 ↓ 或文字 | 待定 |
| 编辑/修改 | ✎ | 铅笔 |

#### SVG 图标使用场景

当 Unicode 符号无法清晰表达含义时，使用 SVG 线性图标：

1. 工具栏底部的「设置」齿轮（如果 ⚙ 不够清晰）
2. 「导出 Diff」按钮的图标
3. 其他需要精致图标的地方

**SVG 图标风格**：
- 线性风格（stroke），非填充
- 统一线宽：1.5px
- 统一尺寸：16px（小按钮）/ 18px（正文）/ 20px（大按钮）
- 颜色：继承当前文字颜色（currentColor）

## 四、实施原则

> **核心原则**：按参考设计的标准迭代每一个元素的设计，而不是只关注颜色。

1. **视觉细节**：每一个像素的圆角、阴影、间距都要对齐设计
2. **交互细节**：每一个 hover/active/focus 状态都要有明确的视觉反馈
3. **状态完整**：每一个组件都要有 default/hover/active/disabled 四态
4. **可访问性**：键盘可达、语义正确、对比度达标
5. **性能**：动画流畅，无重绘性能问题
