## 二、工具栏差异详情 (Toolbar)

### 差异 T-1：工具栏控制按钮图标形式（SVG vs 文字符号）
- **演示文件**：`hdm-toolbar-window-ctrl` 中只有 2 个按钮（最小化 + 关闭），使用文字符号：最小化(-)、关闭(×)
- **当前实现**：顺序一致（最小化 + 关闭），但按钮使用 SVG 图标，而非文字符号
- **位置**：`content.js` 第 2988-3006 行
- **严重程度**：P1（中）
- **根因**：使用 SVG 图标库替代文字符号，视觉上略有不同
- **用户决策**：用户已确认工具栏控制按钮改成文字符号（而非SVG图标）

### 差异 T-2：底部快捷键提示样式
- **演示文件**：`hdm-shortcut` 中快捷键使用 `<kbd>` 标签包裹，样式为白底灰边圆角
- **当前实现**：使用 `html-diff-marker-kbd` 类，样式基本一致
- **位置**：`content.css` 第 1899-1913 行
- **严重程度**：P3（极低）
- **状态**：基本一致，无重大差异

### 差异 T-3：工具栏阴影
- **演示文件**：`box-shadow: var(--shadow-lg)` → `0 10px 25px rgba(0,0,0,0.1)`
- **当前实现**：`box-shadow: 0 10px 25px rgba(0,0,0,0.1) !important`
- **位置**：`content.css` 第 1654 行

**演示文件中的分组重置按钮形态（按分组区分）**：

| 分组 | 按钮形态 | 位置 | 文字内容 |

## 九、核心差异总结与优先级

### P1 级差异（建议修复）

1. **工具栏控制按钮图标形式**（T-1）：当前使用 SVG 图标，演示文件使用文字符号
   - 用户已确认改成文字符号（而非SVG图标）

2. **编辑面板分组重置按钮样式**（I-3）：
   - 当前所有分组统一为"SVG图标 + 文字"的独立按钮

## 十一、根因分析

1. **设计迭代差异**：当前实现基于 V5 设计进行了独立迭代，部分组件（如编辑面板头部、设置开关）的实现方式与 V5.2 演示方案不同。

2. **功能增强**：当前实现增加了一些功能（如 Toast 关闭按钮、显式的面板关闭按钮），这些是在演示方案基础上的增强。

3. **命名空间差异**：演示文件使用 `hdm-` 前缀，当前实现使用 `html-diff-marker-` 前缀，这是正常的插件样式隔离策略。

4. **实现技术选型不同**：
   - 开关：演示文件用 DOM 元素，当前实现用伪元素
   - Toast 色条：演示文件用伪元素，当前实现用 border
   - 这些都是实现层面的差异，视觉效果基本一致
